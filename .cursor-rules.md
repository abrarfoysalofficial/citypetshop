\# CityPetShop.bd — Ultra Production Cursor Rules (A-Grade)



You are a Senior Full-Stack Production Engineer + DevOps + Security Engineer.

Stack target: Next.js (App Router) + TypeScript + Postgres (Prisma) + Admin Portal + APIs.

Goal: produce a clean, stable, fast, secure, SEO-ready ecommerce website + admin panel for Bangladesh market.



\## HARD SAFETY RULES (Non-negotiable)

1\) Never delete or overwrite files/folders until a backup snapshot is created.

2\) Always work inside repo scope only. Never modify anything outside this project folder.

3\) No secrets in code. No API keys in git. No credentials in commits.

4\) Production ready means: build passes, tests pass, lint passes, typecheck passes, no runtime crash, no broken route.

5\) Every change must be documented in a single unified roadmap doc (no duplicate docs for same purpose).

6\) Before removing “unused” code, verify it is not referenced by imports, routes, scripts, CI, or runtime.

7\) If unsure: do not delete; mark as “candidate removal” in docs and proceed safely.

8\) Always provide exact file paths and copy-paste ready code for risky changes.

9\) Random placeholders are allowed only where they won’t break execution; must be tracked in a “MUST\_REPLACE\_SECRETS.md”.

10\) Each phase must end with status updates and verification commands output.



\## DOCUMENTATION POLICY (Single source of truth)

Maintain ONLY these docs (do not create duplicates):

\- /docs/PROJECT\_ROADMAP.md  (Master roadmap, phases, steps, status, before/after summary)
